# Промт: Приложение прогнозов на матчи РПЛ 2026/2027

## Общее описание

Самостоятельное веб-приложение для прогнозирования результатов матчей Российской Премьер-Лиги сезона 2026/2027. Пользователи регистрируются с логином и паролем, делают прогнозы на матчи, набирают очки за правильные прогнозы и соревнуются в таблице лидеров. Приложение создаётся как новый проект с нуля.

## Технологический стек

- **Frontend:** React 19 + React Router 7 + Vite 8
- **Backend/БД:** Supabase (PostgreSQL + Realtime + REST API)
- **Хостинг:** Vercel (или аналогичный)
- **PWA:** vite-plugin-pwa + Workbox
- **Стейт-менеджмент:** React Context + hooks
- **Стили:** CSS (без фреймворков)
- PWA
- Push Notifications
- адаптивная мобильная версия
- версификация через sw. при нахождении новой версии, предлагать обновить.

## Структура чемпионата

- **Формат:** Круговая система (двойной круг)
- **Команды:** 16
- **Туры:** 30
- **Матчей в туре:** 8 (16 команд / 2)
- **Всего матчей:** 480 (30 × 16)

## Авторизация

- **Тип:** Логин + пароль (без email, без OAuth)
- **Регистрация:** Ввод логина (мин. 3 символа) + пароля (мин. 6 символов)
- **Хеширование:** SHA-256 через расширение PostgreSQL `pgcrypto`
- **Сессия:** Хранится в localStorage, восстанавливается при загрузке
- **Выход:** Очистка localStorage

### SQL функции

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE OR REPLACE FUNCTION register_user(login TEXT, password TEXT)
RETURNS UUID AS $$
DECLARE
  new_id UUID;
BEGIN
  INSERT INTO users (username, password_hash)
  VALUES (login, encode(digest(password, 'sha256'), 'hex'))
  RETURNING id INTO new_id;
  RETURN new_id;
END;
$$ LANGUAGE plpgsql;

CREATE OR REPLACE FUNCTION verify_user(login TEXT, password TEXT)
RETURNS TABLE(id UUID, username TEXT, created_at TIMESTAMPTZ) AS $$
BEGIN
  RETURN QUERY
  SELECT u.id, u.username, u.created_at
  FROM users u
  WHERE u.username = login
    AND u.password_hash = encode(digest(password, 'sha256'), 'hex');
END;
$$ LANGUAGE plpgsql;
```

## Система начисления очков

### Типы прогнозов

| Тип | Описание | Очки |
|---|---|---|
| Точный счёт | Угаданы оба числа (например, 2:1) | 5 |
| Исход (1/X/2) | Угадан результат матча (победа/ничья/поражение) | 3 |
| Порог голов | Команда А забьёт не менее X голов | X (равно порогу) |

### Правила суммирования

- Точный счёт + Исход = 5 + 3 = 8 очков (максимум за матч)
- Порог голов + Исход = X + 3 очков
- Порог голов НЕ суммируется с точным счётом
- Исход засчитывается и при точном счёте, и при пороге, и отдельно

### Примеры

| Прогноз | Реальность | Очки |
|---|---|---|
| 2:1 | 2:1 | 8 (5+3) |
| 2:1 | 2:0 | 3 (только исход) |
| П1 (без счёта) | 3:1 | 3 |
| Команда ≥ 2 голов | Забила 3 | 2 |
| Команда ≥ 2 голов + П1 | Забила 3, победа | 5 (2+3) |
| Команда ≥ 3 голов | Забила 2 | 0 |

### Валидация

- Порог голов: минимум 1 гол
- Точный счёт: целые неотрицательные числа
- Исход: 1, X или 2


### Таблица predictions

```sql
CREATE TABLE predictions (
  id SERIAL PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  match_id INTEGER NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
  predicted_home_score INTEGER,
  predicted_away_score INTEGER,
  outcome TEXT CHECK (outcome IN ('1', 'X', '2')),
  goals_team TEXT CHECK (goals_team IN ('home', 'away')),
  goals_threshold INTEGER,
  points_earned INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(user_id, match_id)
);
```

### Функция calculate_points

```sql
CREATE OR REPLACE FUNCTION calculate_points(
  predicted_home INTEGER,
  predicted_away INTEGER,
  actual_home INTEGER,
  actual_away INTEGER,
  outcome TEXT,
  goals_team TEXT,
  goals_threshold INTEGER
) RETURNS INTEGER AS $$
DECLARE
  points INTEGER := 0;
  predicted_outcome TEXT;
  actual_outcome TEXT := CASE
    WHEN actual_home > actual_away THEN '1'
    WHEN actual_home = actual_away THEN 'X'
    ELSE '2'
  END;
BEGIN
  IF predicted_home IS NOT NULL AND predicted_away IS NOT NULL THEN
    IF predicted_home = actual_home AND predicted_away = actual_away THEN
      points := points + 5;
    END IF;

    predicted_outcome := CASE
      WHEN predicted_home > predicted_away THEN '1'
      WHEN predicted_home = predicted_away THEN 'X'
      ELSE '2'
    END;
    IF predicted_outcome = actual_outcome THEN
      points := points + 3;
    END IF;
  END IF;

  IF outcome IS NOT NULL AND (predicted_home IS NULL OR predicted_away IS NULL) THEN
    IF outcome = actual_outcome THEN
      points := points + 3;
    END IF;
  END IF;

  IF goals_team IS NOT NULL AND goals_threshold IS NOT NULL THEN
    IF (goals_team = 'home' AND actual_home >= goals_threshold)
       OR (goals_team = 'away' AND actual_away >= goals_threshold)
    THEN
      points := points + goals_threshold;
    END IF;
  END IF;

  RETURN points;
END;
$$ LANGUAGE plpgsql;
```

### Триггер

```sql
CREATE OR REPLACE FUNCTION recalculate_match_predictions() RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = 'FINISHED' THEN
    UPDATE predictions
    SET points_earned = calculate_points(
      predicted_home_score,
      predicted_away_score,
      NEW.home_score,
      NEW.away_score,
      outcome,
      goals_team,
      goals_threshold
    )
    WHERE match_id = NEW.id;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

DROP TRIGGER IF EXISTS trigger_recalculate_predictions ON matches;
CREATE TRIGGER trigger_recalculate_predictions
  AFTER UPDATE OF home_score, away_score, status ON matches
  FOR EACH ROW
  WHEN (
    OLD.status IS DISTINCT FROM NEW.status
    OR OLD.home_score IS DISTINCT FROM NEW.home_score
    OR OLD.away_score IS DISTINCT FROM NEW.away_score
  )
  EXECUTE FUNCTION recalculate_match_predictions();
```

### Представление leaderboard

```sql
CREATE OR REPLACE VIEW leaderboard AS
SELECT
  u.id,
  u.username,
  COALESCE(SUM(p.points_earned), 0)::INTEGER AS total_points,
  COUNT(p.id)::INTEGER AS total_predictions,
  COUNT(CASE WHEN p.points_earned >= 5 THEN 1 END)::INTEGER AS exact_scores,
  COUNT(CASE WHEN p.points_earned >= 3 THEN 1 END)::INTEGER AS correct_outcomes,
  COUNT(CASE WHEN p.points_earned > 0 THEN 1 END)::INTEGER AS scored_predictions
FROM users u
LEFT JOIN predictions p ON u.id = p.user_id
GROUP BY u.id, u.username
ORDER BY total_points DESC;
```
